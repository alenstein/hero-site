Docs chevron_right Authentication

authentication

To access the thetha API, you need to provide an API key in the Authorization header of your requests.

api key security

Keep your API key secure and do not share it in client-side code. If your key is compromised, revoke it immediately through the dashboard to prevent unauthorized usage.

verified_user

Security Best Practice

Always use environment variables for keys. Never commit .env files to public repositories.

authorization types

Bearer Token Default

Standard authentication using the Bearer scheme. Recommended for most integrations.

curl -X GET "https://api.thetha.ai/v1/auth" \
     -H "Authorization: Bearer YOUR_API_KEY"
